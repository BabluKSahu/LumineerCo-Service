2. PLATFORM STRUCTURE: DUAL-USER HOMEPAGE DESIGN

Homepage Differentiation Strategy

Your homepage must have two distinct entry points to serve customers and agents with different value propositions.

Element: Primary CTA
Customer (User) Homepage: "Book a Service Now"
Agent (Service Provider) Homepage: "Become an Agent & Earn"

Element: Hero Message
Customer (User) Homepage: "Professional Website, App & Repair Services at Your Doorstep"
Agent (Service Provider) Homepage: "Grow Your Service Business with LumineerCo - Zero Commission First 30 Days"

Element: Key Features Highlighted
Customer (User) Homepage: Service categories, pricing, agent ratings, reviews, booking process
Agent (Service Provider) Homepage: Lead volume, earning potential, commission structure, support, verification process

Element: Trust Signals
Customer (User) Homepage: Customer testimonials, completed projects, agent ratings (5-star avg)
Agent (Service Provider) Homepage: Agent success stories, payment proof, withdrawal history, training resources

Element: Call-to-Action Flow
Customer (User) Homepage: Select Service -> Fill Details -> Book Slot -> Pay Rs.300 Booking Fee -> Confirm -> Receive Agent Assignment
Agent (Service Provider) Homepage: Register Profile -> Submit Documents -> Verification (24-48 hrs) -> Activate -> Receive Leads -> Start Earning

Element: Navigation Menu
Customer (User) Homepage: Services, Pricing, Reviews, FAQ, Contact, My Bookings, Warranty Claims
Agent (Service Provider) Homepage: Dashboard, My Leads, Earnings, Withdrawal, Support, Training, Profile

Element: Mobile Optimization
Customer (User) Homepage: One-tap booking, real-time agent tracking, instant notifications
Agent (Service Provider) Homepage: Lead notifications, quick acceptance, photo/video upload, instant billing

3. DETAILED USER FLOW (CUSTOMERS)

Step-by-Step Process with Revenue Touchpoints

Step 1: Service Selection
Process: Customer visits website/app and selects service (Website Dev, App Dev, Device Repair)
Platform Action: Display service categories with pricing tiers
Revenue/Data Capture: -
Timeline: Immediate

Step 2: Requirement Details
Process: Customer fills form: Service type, description, location (Noida/NCR), contact, preferred date
Platform Action: Validate location and check agent availability
Revenue/Data Capture: -
Timeline: 5-10 minutes

Step 3: Booking Fee Payment
Process: Customer pays non-refundable Rs.300 booking fee (GST applicable: Rs.354 total)
Platform Action: Capture payment via Razorpay/PayU, send confirmation SMS + Email
Revenue/Data Capture: Rs.300 revenue per booking
Timeline: Immediate

Step 4: Slot Confirmation
Process: System assigns available slot; customer receives booking reference ID
Platform Action: Send SMS + Email with confirmation, agent name, contact details
Revenue/Data Capture: Booking logged in database
Timeline: Within 1 hour

Step 5: Agent Assignment
Process: Platform assigns agent based on availability, rating, and location proximity
Platform Action: Notify agent of new lead; charge agent Rs.500-Rs.1,000 lead fee from account balance
Revenue/Data Capture: Rs.500-Rs.1,000 revenue (lead charge)
Timeline: Within 2 hours

Step 6: Customer-Agent Coordination
Process: Customer and agent communicate via platform messaging/call to confirm visit date
Platform Action: Enable in-app messaging, call logging, and document sharing
Revenue/Data Capture: Track communication for quality assurance
Timeline: 24-48 hours

Step 7: Agent Visit & Assessment
Process: Agent visits location, assesses work, and updates report on platform
Platform Action: Agent uploads photos, work description, required items, and estimated hours
Revenue/Data Capture: Visiting charge: Rs.500 (first 2 visits free; Rs.250/km beyond 10km)
Timeline: Within 48 hours

Step 8: Estimate Bill Sharing
Process: Agent prepares and shares estimate bill on platform
Platform Action: Bill includes: labor cost + materials + GST (18%); platform commission: 10-15% of bill
Revenue/Data Capture: Rs.500-Rs.5,000 commission per project
Timeline: Within 24 hours

Step 9: Customer Approval
Process: Customer reviews estimate, asks clarifications, approves/rejects
Platform Action: Enable bill modification requests and approval workflow
Revenue/Data Capture: -
Timeline: 24 hours

Step 10: Service Execution
Process: Agent completes work, uploads photos/videos on platform
Platform Action: Create work progress tracker; customer can monitor updates
Revenue/Data Capture: -
Timeline: 1-30 days (depends on service)

Step 11: Final Bill & Payment
Process: Agent creates final bill; customer pays via platform (Razorpay/PayU)
Platform Action: Payment split: 85% to agent, 15% to platform (commission + GST)
Revenue/Data Capture: Final commission: 15% of total service cost
Timeline: Upon completion

Step 12: Warranty Registration
Process: Agent enables warranty on platform; customer receives warranty certificate
Platform Action: Store warranty details in customer account; set expiry reminders
Revenue/Data Capture: -
Timeline: Upon completion

Step 13: Feedback & Rating
Process: Customer rates agent (1-5 stars) and leaves review
Platform Action: Update agent profile with rating; use for agent ranking and trust building
Revenue/Data Capture: Improve agent visibility and future bookings
Timeline: Within 7 days

Step 14: Warranty Claim
Process: If issue arises, customer uploads work photos/videos and requests warranty service
Platform Action: Create warranty claim ticket; assign to original agent or similar agent
Revenue/Data Capture: -
Timeline: As needed

4. DETAILED AGENT FLOW (SERVICE PROVIDERS)

Recruitment, Verification & Earning Structure

Step 1: Agent Registration
Process: Agent signs up via website/app; provides name, phone, email, location, service category
Platform Action: Capture basic info; generate unique agent ID
Revenue/Cost to Agent: -
Timeline: Immediate

Step 2: Document Submission
Process: Agent uploads: Aadhar, PAN, Bank details, Service certificates, Portfolio/previous work
Platform Action: Store documents securely; trigger verification workflow
Revenue/Cost to Agent: -
Timeline: 24 hours

Step 3: Verification & Approval
Process: Admin reviews documents; conducts background check (if needed)
Platform Action: Mark agent as "Verified" or "Pending"; send approval email
Revenue/Cost to Agent: -
Timeline: 24-48 hours

Step 4: Account Setup & Top-Up
Process: Agent adds wallet balance (minimum Rs.1,000) via bank transfer/UPI
Platform Action: Create wallet; enable lead purchase functionality
Revenue/Cost to Agent: Agent invests initial capital
Timeline: Upon approval

Step 5: Lead Assignment
Process: Customer books service; system automatically assigns agent based on rating/availability
Platform Action: Deduct Rs.500-Rs.1,000 lead fee from agent's wallet balance
Revenue/Cost to Agent: Agent pays Rs.500-Rs.1,000 per lead
Timeline: Automatic

Step 6: Lead Acceptance
Process: Agent reviews lead details and accepts/rejects within 30 minutes
Platform Action: If rejected, lead reassigned to another agent
Revenue/Cost to Agent: Lead fee deducted regardless
Timeline: Within 30 minutes

Step 7: Customer Coordination
Process: Agent contacts customer via in-app messaging/call to confirm visit date/time
Platform Action: Log all communications on platform
Revenue/Cost to Agent: -
Timeline: Within 6 hours

Step 8: First Visit
Process: Agent visits customer location, assesses work, takes photos
Platform Action: Upload assessment report, required materials list, estimated hours
Revenue/Cost to Agent: Free first visit (Rs.0 charge)
Timeline: Within 48 hours

Step 9: Second Visit (if needed)
Process: Agent returns for follow-up or additional assessment
Platform Action: Upload updated report
Revenue/Cost to Agent: Free second visit (Rs.0 charge)
Timeline: Within 48 hours

Step 10: Additional Visits
Process: Any visit beyond 2 visits charged to customer
Platform Action: Calculate distance; charge customer Rs.250/km for distance > 10km
Revenue/Cost to Agent: Agent receives 80% of visiting charge
Timeline: As needed

Step 11: Estimate Bill Creation
Process: Agent prepares itemized estimate bill on platform
Platform Action: Bill breakdown: Labor (Rs.X) + Materials (Rs.Y) + GST (18%) + Platform Commission (10-15%)
Revenue/Cost to Agent: Agent sees net amount after commission
Timeline: Within 24 hours

Step 12: Bill Approval Tracking
Process: Agent monitors bill approval status; customer can request modifications
Platform Action: Enable back-and-forth negotiation on platform
Revenue/Cost to Agent: -
Timeline: 24 hours

Step 13: Service Execution
Process: Agent completes work; uploads photos/videos as proof of completion
Platform Action: Create work completion checklist
Revenue/Cost to Agent: -
Timeline: 1-30 days

Step 14: Final Bill & Payment Collection
Process: Agent creates final bill; customer pays via platform
Platform Action: Payment processed: 85% to agent, 15% to platform
Revenue/Cost to Agent: Agent receives 85% of service cost
Timeline: Upon completion

Step 15: Warranty Assignment
Process: Agent marks warranty period and terms on platform
Platform Action: System auto-generates warranty certificate
Revenue/Cost to Agent: -
Timeline: Upon completion

Step 16: Performance Rating
Process: Customer rates agent; rating impacts future lead assignment priority
Platform Action: Update agent rating; boost visibility if 4.5+ stars
Revenue/Cost to Agent: High-rated agents get priority leads
Timeline: Within 7 days

Step 17: Earnings Dashboard
Process: Agent can view total earnings, completed jobs, pending payments, withdrawal history
Platform Action: Real-time earnings tracker; monthly payouts via bank transfer
Revenue/Cost to Agent: -
Timeline: Ongoing

Step 18: Withdrawal & Payout
Process: Agent withdraws earnings via bank transfer (minimum Rs.500, max Rs.50,000/day)
Platform Action: Process withdrawal within 24-48 hours; deduct 2% transaction fee
Revenue/Cost to Agent: Agent keeps 98% of earnings
Timeline: On demand

TABLE 1: users (Customers)
CREATE TABLE users (
    user_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    first_name VARCHAR(100) NOT NULL,
    last_name VARCHAR(100) NOT NULL,
    email VARCHAR(255) UNIQUE NOT NULL,
    phone_number VARCHAR(15) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    profile_picture_url VARCHAR(500),
    location_city VARCHAR(100) NOT NULL,
    location_area VARCHAR(100) NOT NULL,
    location_pincode VARCHAR(10) NOT NULL,
    location_latitude DECIMAL(10, 8),
    location_longitude DECIMAL(11, 8),
    address_full_text TEXT,
    account_status ENUM('active', 'inactive', 'suspended') DEFAULT 'active',
    email_verified BOOLEAN DEFAULT FALSE,
    phone_verified BOOLEAN DEFAULT FALSE,
    kyc_status ENUM('not_started', 'pending', 'verified', 'rejected') DEFAULT 'not_started',
    kyc_document_url VARCHAR(500),
    total_bookings INT DEFAULT 0,
    total_spent DECIMAL(12, 2) DEFAULT 0.00,
    average_rating DECIMAL(3, 2) DEFAULT 0.00,
    preferred_payment_method VARCHAR(50),
    wallet_balance DECIMAL(10, 2) DEFAULT 0.00,
    referral_code VARCHAR(20) UNIQUE,
    referred_by_user_id UUID REFERENCES users(user_id),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    last_login TIMESTAMP,
    device_token VARCHAR(500)
);
